# 前端操作说明

## 页面入口

| 页面 | 路由 | 当前用途 |
|---|---|---|
| 路网大屏 | `/` | 路网展示、仿真控制、告警、信号面板、应急调度和 Agent |
| 数据分析 | `/data-analysis` | 展示后端 bootstrap 返回的数据分析内容 |
| 登录 | `/login` | 用户名密码或邮箱验证码登录 |
| 注册 | `/register` | 使用邀请码注册账号 |

当前路由没有前端登录守卫。进入登录页或取得 token 不会限制其他页面和业务接口。

## 路网大屏

主要区域包括 `TrafficStats`、`AlertPanel`、`MapRoadNetwork`、`SignalControlPanel`、`EmergencyPanel`、`CompareCharts` 和 `AiAssistant`。

页面先调用 `GET /api/v1/dashboard/bootstrap`。成功时显示“数据库数据”，这里表示 bootstrap 数据由后端数据库展示层返回，不等于实时 CityFlow 帧；失败时使用本地 mock 并显示“本地演示”。仿真创建并启动后，实时帧通过 WebSocket 更新路网状态。

## 当前操作

- AI 启动、暂停和手动切换相位只修改前端 store，不下发真实控制。
- 单击告警会将其标记为已确认，并在有关联路口时选中该路口。
- “全部确认”会确认当前 store 中全部未确认告警。
- “应急调度”可选择起终点、车辆类型和优先级。存在仿真 `sid` 时优先调用后端；否则或调用失败时使用本地网格路线。
- Agent 只调用 Spring Boot 的 `POST /api/v1/agent/chat`，前端没有本地问答兜底。

页面中的展示值、策略对比和本地 mock 不能作为真实路口运行或控制效果的证据。
