# 应急任务创建流程

## 文档信息

- 知识库分类：系统操作手册知识库
- 来源条目：应急任务创建流程
- 前端入口：路网大屏右侧“应急绿波控制”面板

- 当前版本边界：前端已支持应急车辆进入路网和绿波执行演示。后端已有应急调度接口结构，但完整任务创建、审批、持久化和真实控制下发仍需进一步接入。

## 应急任务定义

应急任务是指救护车、消防车等应急车辆进入路网后，系统为其规划优先通行路线，并在沿线路口形成绿波窗口的调度流程。

当前系统中，应急任务主要用于演示：

1. 应急车辆进入路网。
2. 系统识别起点、终点和路线节点。
3. 沿线绿波路口逐步激活。
4. 展示预计到达时间和预计节省时间。
5. 任务结束后恢复普通控制。

## 任务状态

前端应急面板使用以下状态：

| 状态 | 含义 |
|---|---|
| 未触发 | 当前没有应急事件或绿波任务 |
| 绿波规划中 | 已有应急车辆，但绿波尚未执行，保留给后续后端任务接入 |
| 绿波执行中 | 应急车辆和绿波通道处于激活状态 |
| 已完成 | 应急流程结束，系统回到普通控制 |

当前前端按钮“模拟救护车进入路网”会直接进入应急绿波执行态。

## 创建演示任务流程

### 1. 打开路网大屏

进入 `/`，确认右侧存在“应急绿波控制”面板。

### 2. 检查当前状态

如果显示“路网运行正常，无应急事件”，表示当前处于未触发状态。

### 3. 模拟应急车辆进入路网

点击“模拟救护车进入路网”。

系统会执行以下演示动作：

- 将应急车辆 `E001` 设置为救护车。
- 将应急车辆放置在应急路线首段。
- 设置系统模式为 `emergency`。
- 激活绿波通道。
- 生成“应急绿波通道已激活”告警。
- 在面板中显示车辆 ID、路线、速度、ETA 和节省时间。

### 4. 查看任务信息

面板会显示：

- 应急车辆类型。
- 车辆 ID。
- 起点。
- 终点。
- 目的地方向描述。
- 当前速度。
- 预计到达。
- 预计节省时间。
- 绿波路线节点。
- 已激活路口数量。
- 绿波进度百分比。

### 5. 观察路线进度

当车辆在路网上移动时，系统会根据车辆进度推进绿波节点。

绿波路线节点会以链式形式展示，例如：

```text
A01 -> A02 -> A03 -> A04
```

当前节点会高亮，已放行节点会显示激活状态。

## 后端任务接口边界

后端已预留应急调度接口：

```text
POST /api/v1/simulations/{sid}/dispatch
```

该接口用于在指定仿真会话中发起应急车辆调度。完整业务接入后，应急任务应包含：

- 仿真会话 ID。
- 应急车辆类型。
- 起点坐标或起点路口。
- 终点坐标或目的地。
- 任务优先级。
- 预计路线。
- 审批状态。
- 创建人。
- 创建时间。
- 执行状态。
- 审计记录。

当前前端面板尚未把该后端接口完整接入到任务创建流程。

## 任务创建建议规范

正式接入后，建议应急任务按以下步骤创建：

1. 调度员填写应急任务草案。
2. 系统生成推荐路线和绿波方案。
3. 应急管理员确认任务。
4. 系统启动绿波执行。
5. 系统记录任务状态和操作日志。
6. 任务完成后生成复盘报告。

## 人工确认点

以下环节应要求人工确认：

- 是否创建真实应急任务。
- 是否启用沿线绿波。
- 是否覆盖当前路口配时。
- 是否恢复普通控制。
- 任务完成后是否关闭优先通道。
