# 应急任务创建流程

## 前置条件

入口位于路网大屏右侧“应急绿波控制”。后端接口是 `POST /api/v1/simulations/{sid}/dispatch`，要求仿真会话处于 running 状态。

## 操作流程

1. 点击“应急调度”。
2. 选择不同的起点和终点。
3. 选择车辆类型和优先级。
4. 确认调度。
5. 查看返回或本地生成的路线、车辆信息、预计到达时间和绿波进度。
6. 需要时使用“新增应急车辆”“启动应急绿波”或“恢复普通控制”。

如果前端已有仿真 `sid`，会优先调用后端调度接口。没有 `sid` 或接口失败时，前端会生成本地 Manhattan 网格路线并进入演示状态。界面出现路线不代表后端任务一定创建成功，应结合数据来源和后端日志判断。

## 当前状态

面板状态包括未触发、绿波规划中、绿波执行中和已完成。成功的后端调度与本地回落都会把 `systemMode` 设为 `emergency`、激活绿波并显示车辆。完整的生产审批、角色授权和真实信号机控制不在当前实现中。
