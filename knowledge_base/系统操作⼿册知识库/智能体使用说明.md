# 智能体使用说明

## 文档信息

- 知识库分类：系统操作手册知识库
- 来源条目：智能体使用说明
- 前端入口：路网大屏右下角“AI 决策”悬浮按钮
- 后端接口：`POST /api/v1/agent/chat`
- 当前版本边界：智能体支持百炼后端代理和本地兜底规则。涉及控制动作时，智能体只应给出建议和待确认方案，不应直接执行真实控制。
- 相关文件：
  - `sys-frontend/src/components/AiAssistant.vue`
  - `backend/src/main/java/com/traffic/agent/controller/AgentController.java`
  - `backend/src/main/java/com/traffic/agent/service/BailianAgentService.java`

## 智能体定位

智能体是交通信号调度辅助决策入口，用于回答路网状态、拥堵原因、应急路线、信号配时建议和策略解释等问题。

智能体不应绕过人工确认直接执行真实控制。当前系统中，智能体主要承担：

1. 状态查询解释。
2. 拥堵原因分析。
3. 信号配时建议生成。
4. 应急绿波路线说明。
5. Traffic-R1、MaxPressure 等策略解释。
6. 失败时使用本地兜底规则回答常见问题。

## 打开与关闭

1. 在路网大屏右下角点击“AI 决策”。
2. 弹出“智能体辅助决策”窗口。
3. 点击右上角关闭按钮可关闭窗口。
4. 点击“新对话”可清空当前会话并重新开始。

## 推荐问题

智能体窗口提供快捷提问：

- 当前路网状态
- 哪个路口最拥堵？
- 生成调度建议
- 应急车辆怎么走？
- 解释 Traffic-R1

调度人员也可以手动输入问题。

## 可询问的问题类型

| 类型 | 示例问题 | 预期回答 |
|---|---|---|
| 路网态势 | 当前路网状态怎么样？ | 汇总车流、速度、等待时间、拥堵指数 |
| 拥堵定位 | 哪个路口最拥堵？ | 给出拥堵最高路口、排队、延误和建议 |
| 信号建议 | 为什么要延长绿灯？ | 解释排队、拥堵指数和绿灯延长理由 |
| 应急通行 | 应急车辆怎么走？ | 给出应急路线、绿波窗口和预计到达 |
| 设备异常 | 哪些设备离线？ | 列出离线或故障路口并建议巡检 |
| 策略解释 | Traffic-R1 和 MaxPressure 有什么区别？ | 从策略原理和适用边界解释 |

## 智能体请求内容

前端会把以下内容发送给后端：

- 用户输入的问题 `message`
- 当前会话 ID `sessionId`
- 当前路网上下文 `context`

上下文包括：

- 全局统计指标。
- 拥堵最高的前 5 个路口。
- 道路数量。
- 应急车辆状态。
- 应急路线。
- 控制效果对比指标。

## 智能体状态说明

| 状态 | 含义 |
|---|---|
| 百炼待连接 | 尚未完成有效百炼会话，或刚打开窗口 |
| 百炼在线 | 后端百炼代理正常返回 |
| 配置待完成 | 后端返回了配置兜底结果 |
| 本地兜底 | 前端未能连接百炼或后端异常，改用本地规则 |

## 本地兜底能力

当百炼接口不可用时，前端会尝试用本地规则回答以下问题：

- 最拥堵路口。
- 为什么延长绿灯。
- 应急车辆路线。
- 设备离线或故障。

如果本地规则也无法覆盖，会提示检查：

- 后端服务是否启动。
- `BAILIAN_API_KEY` 是否配置。
- 百炼应用 ID 是否配置。
- 网络或代理是否可用。

## 使用规范

### 可以直接询问

- 当前哪个路口最拥堵？
- 当前平均等待时间是多少？
- 应急车辆预计多久到达？
- 设备离线应该如何处理？
- 当前 AI 建议依据是什么？

### 需要人工确认后再执行

- 启动应急绿波。
- 延长某方向绿灯。
- 手动切换相位。
- 暂停 AI 控制。
- 恢复普通控制。

智能体回答中如出现“建议”“待确认”“需人工确认”，表示该内容不能直接等同于已执行。

## 推荐提问格式

为了让智能体更准确，可以包含路口、时间、指标或任务背景。

示例：

```text
请分析当前拥堵指数最高的路口，并给出是否需要延长绿灯的建议。
```

```text
当前有救护车进入路网，请说明推荐绿波路线、预计节省时间和人工确认点。
```

```text
解释为什么系统建议对当前路口暂停 AI 控制。
```

## 输出解释口径

智能体回答应区分三类结果：

| 类型 | 说明 |
|---|---|
| 状态信息 | 来自当前页面上下文或后端数据，可用于观察 |
| 调度建议 | 可供调度员参考，需要人工判断 |
| 控制动作 | 需要权限、审批、确认和审计，不由智能体直接执行 |

## 适合回答的问题

- 智能体入口在哪里？
- 智能体能回答什么？
- 百炼不可用时怎么办？
- 智能体会不会直接控制信号灯？
- 智能体上下文包含哪些数据？
- 如何开启新对话？
- 智能体生成的建议是否需要人工确认？
