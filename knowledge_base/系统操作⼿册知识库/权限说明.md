## 权限设计目标

权限说明用于让系统和智能体知道“谁能做什么”。它不只描述页面能不能看，还要描述建议生成、方案确认、控制执行、审计查看等操作边界。

当前系统应按照“查询、建议、确认、执行、审计”分层理解权限。

## 推荐角色

| 角色 | 定位 |
|---|---|
| 观察员 | 只查看路网、指标、告警和报告，不生成或确认控制方案 |
| 调度员 | 可以查看状态、生成建议、处理普通告警和发起草案 |
| 应急管理员 | 可以确认应急绿波方案、恢复普通控制、查看应急复盘 |
| 系统管理员 | 管理系统配置、用户权限、审计记录和服务状态 |

## 推荐权限矩阵

| 操作 | 观察员 | 调度员 | 应急管理员 | 系统管理员 |
|---|---:|---:|---:|---:|
| 查看路网大屏 | 是 | 是 | 是 | 是 |
| 查看数据分析 | 是 | 是 | 是 | 是 |
| 查看交通统计 | 是 | 是 | 是 | 是 |
| 查看实时告警 | 是 | 是 | 是 | 是 |
| 确认普通告警 | 否 | 是 | 是 | 是 |
| 全部确认告警 | 否 | 部分 | 是 | 是 |
| 查看 AI 建议 | 是 | 是 | 是 | 是 |
| 生成调度建议 | 否 | 是 | 是 | 是 |
| 启动 AI 自适应控制 | 否 | 部分 | 是 | 是 |
| 暂停 AI 控制 | 否 | 部分 | 是 | 是 |
| 手动切换相位 | 否 | 部分 | 是 | 是 |
| 创建应急任务草案 | 否 | 是 | 是 | 是 |
| 确认应急绿波执行 | 否 | 否 | 是 | 是 |
| 恢复普通控制 | 否 | 否 | 是 | 是 |

## 当前系统中的权限边界

当前前端页面还没有登录态和角色态，因此页面按钮不会按角色隐藏。

当前所有按钮应理解为：

- 可用于本地演示。
- 可用于知识库和操作流程说明。
- 不能直接视为生产权限已经开放。
- 真实控制接入前必须增加鉴权、审批和审计。

## 智能体权限边界

智能体可以：

- 回答路网状态。
- 解释拥堵原因。
- 生成调度建议。
- 说明应急路线。
- 总结告警处理建议。

智能体不应直接：

- 下发真实信号控制。
- 抢占路口相位。
- 确认应急任务执行。
- 恢复真实控制模式。
- 修改用户权限。
- 删除或篡改审计日志。

当用户提出控制类请求时，智能体应回答“可生成建议，需人工确认”。

## 操作确认等级

建议将操作分为四级：

| 等级 | 操作类型 | 确认要求 |
|---|---|---|
| L0 查看 | 查询状态、查看报表、查看告警 | 无需确认 |
| L1 建议 | 生成调度建议、解释原因 | 无需审批，但要标注建议来源 |
| L2 演示控制 | 前端演示按钮、仿真任务操作 | 调度员确认 |
| L3 真实控制 | 真实信号下发、应急绿波执行、恢复普通控制 | 应急管理员或系统管理员确认，并写审计 |

## 审计建议

以下操作建议写入审计日志：

- 启动或暂停 AI 控制。
- 手动切换相位。
- 创建应急任务。
- 确认应急绿波。
- 恢复普通控制。
- 批量确认告警。
- 修改设备状态。
- 修改权限或系统配置。
- 调用智能体生成控制建议。

审计字段建议包括：

- 操作人。
- 角色。
- 操作时间。
- 操作类型。
- 操作对象。
- 操作前状态。
- 操作后状态。
- 是否人工确认。
- 智能体建议 ID。
- 备注。

## 适合回答的问题

- 谁可以查看路网大屏？
- 谁可以确认应急绿波？
- 调度员能不能恢复普通控制？
- 智能体能不能直接执行控制？
- 当前系统是否已经接入权限？
- 哪些操作需要审计？
