# 项目简要介绍

## 项目名称

AI 自适应信号控制与应急绿波数字孪生系统。

## 要解决的问题

1. 传统固定信号灯难以适应实时车流变化，容易在高峰、事故或局部拥堵时产生排队扩散。
2. 多路口之间缺乏协同控制，单点优化难以兼顾干线路网整体通行效率。
3. 救护车、消防车等应急车辆通行效率不足，沿线信号缺少提前联动和动态让行能力。

## 项目概述

本项目面向城市路网信号控制场景，构建一个可视化、可仿真、可扩展的交通数字孪生系统。系统以 Vue 前端大屏展示道路、路口、车辆、信号灯、拥堵状态和应急路线，以 Spring Boot 作为统一业务后端，以 Python CityFlow 服务承载仿真和算法计算。短期目标是打通路网加载、仿真会话、实时车辆动画和指标展示；后续逐步接入 AI 自适应信号控制、应急绿波、智能体辅助决策和 Traffic-R1 / MaxPressure 等算法评测闭环。

## 核心功能

### 交通数据分析大屏

系统在前端构建城市道路网络数字孪生视图，展示路口、道路、车辆、信号灯、拥堵指数、告警事件和系统运行状态。车辆沿道路实时流动，道路根据拥堵程度动态变色，应急车辆路径高亮，沿线路口可进入绿波状态，控制前后指标以图表形式对比。

当前状态：前端主大屏已具备交通主题 mock 演示能力，包括路网渲染、车辆流动、拥堵热力、信号控制面板、应急绿波面板、指标对比和智能体问答面板；后端与 Python 仿真服务正在打通 CityFlow 路网和实时帧主链路。

### AI 自适应信号控制

系统计划根据实时车辆位置、排队长度、等待时间、道路压力和路口状态，动态决定每个路口下一阶段放行方向和持续时间。控制策略通过统一 `TrafficSignalController` 接口接入，后续可切换 FixedTime、MaxPressure、RL、Traffic-R1 等策略。

当前状态：后端已保留 FixedTime、MaxPressure、RL 策略扩展边界，但具体策略尚未接入可视化主链路。Traffic-R1 相关实验资料用于后续评估大模型控制策略，并以 MaxPressure 作为重要 baseline 和工程 fallback。

### 应急绿波控制

当系统检测到救护车、消防车等应急车辆进入路网时，后续将自动创建应急任务，预测到达时间，规划通行路线，并让沿线路口按车辆到达顺序提前放行，形成连续绿波通道。应急结束后，系统恢复普通自适应控制并记录任务复盘指标。

当前状态：前端已提供应急车辆模拟、路线高亮、绿波激活和恢复普通模式的演示交互；后端 `emergency` 包和 `greenwave.*` WebSocket 协议为后续真实任务、路径规划和优先控制预留。

### 智能体辅助决策

系统计划提供自然语言交通调度智能体，支持查询实时路网状态、解释拥堵原因、生成调度建议、回答交通知识库问题，并输出可审计的控制建议。智能体只生成建议和待确认方案，实际信号控制指令仍需经过后端校验和人工确认。

当前状态：前端已有 mock 问答面板和关键词回复；后端 `agent` 包、`agent.*` 消息类型和 RAG 问答边界已预留，后续可接入真实 LLM / RAG 服务。

### 仿真与效果评估

系统通过 CityFlow 路网和流量文件生成仿真状态，实时推送车辆、道路、路口、信号和指标数据。后续将保存场景、路网、会话和指标快照，并用平均排队长度、累计排队车辆数、平均等待时间、平均旅行时间、通行量等指标对比 AI 控制前后效果。

Traffic-R1 计划部分已围绕 Jinan 3x4 路网、LLMTSCS 评测框架、FixedTime / MaxPressure / Random baseline 和 wait-aware 改进方案整理复现资料。该部分暂属独立算法实验，后续稳定后再通过 Python 仿真算法服务接入主系统。

## 技术架构

```text
Vue + TypeScript 前端大屏
  -> Spring Boot REST / WebSocket 主后端
  -> Python CityFlow 仿真与算法服务
  -> CityFlow Engine / Traffic-R1 / MaxPressure 等控制策略
```

主要分工：

| 子系统 | 职责 |
|---|---|
| `sys-frontend` | 大屏可视化、路网/车辆/信号渲染、应急绿波交互、指标对比、智能体面板 |
| `backend` | REST / WebSocket 统一入口、仿真会话、数据库、策略边界、审计和业务编排 |
| `sim-python` | CityFlow 路网解析、仿真会话、实时帧生成，后续承载算法计算 |
| `docs` | 项目结构、通信协议、后端设计、项目介绍和算法资料索引 |
| 外部 `traffic-R1` 工作目录 | Traffic-R1 模型、LLMTSCS 评测、baseline 对比和运行日志 |
